## Easily "Modding" a Website
UI frameworks may seem complicated, but they can make your website look nice with less work. Without frameworks, you would have to create all CSS by yourself. UI frameworks are like aftermarket modifications for a car. For example, an aftermarket exhaust will come with the hardware and pre-bent welded pipes to fit your car, so little to no modifications are needed. If I wanted to create a custom exhaust, then I would have to bend and weld the pipes. Also, I would have to gather the hardware and material to install the custom exhaust. Aftermarket exhaust systems can change the sound of a car with less work. The analogy may be a stretch, but UI frameworks are worth learning to save a lot of time.

## Website Recreation
I tried to recreate a website without semantic UI, and I took a long time to create the menu because I had to use flexbox and grids. On the other hand, I tried to recreate the site with semantic UI, and I just had to add an ui menu class to a div tag, and a bunch of item classes to create a top menu. With only a couple class and item attributes, I was able to create something that resembles a menu. Semantic UI definitely helped me create a nice functional menu in less time than scratch HTML/CSS. 

## Conclusion
I enjoyed using semantic UI because I was able to focus on modifying the UI element instead of crazy HTML/CSS code. Scratch HTML/CSS allows for more customization with different behaviour, but HTML/CSS code can get cluttered and complicated easily. UI frameworks are beneficial for software engineers because they save time for the engineers. The engineers can focus on implementation and customization instead of cluttered HTML/CSS code.
