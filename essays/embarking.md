## Rough Start

I started to code in my AP computer science class, and I was mediocre at solving problems. I was quite indifferent about the class because I accepted the fact
that I just sucked at coding. However, I did not want to waste my parents money (AP classes costed extra), so I studied hard to understand the questions on the AP practice
exams. Sometimes I would just memorize the answers from the past AP exams because they had similar questions. Luckily, I got a 3 on the AP Computer Science exam which allowed
me to be exempt from ICS 111. 

## Reality Struck

I thought I was going to be a mediocre programmer because I barely passed the AP Computer Science exam. Then, in college, I realized that the AP Computer Science class helped
me become a better programmer. ICS 211 was a breeze because I learned all the foundations of Java in AP Computer Science. I developed a passion for programming and I wanted to 
learn more. 

## Multiple Fields of Software Engineering

I knew I wanted to have a career in Software Engineer, but I was unsure of which path to take. There were many paths, such as game development, web application development,
mobile application development, cybersecurity, data science, fullstack web development, etc. I've thought about game development because I enjoyed playing video games. Also, I found
the psycology involved in game reward systems was interesting. There was a high demand for web and mobile application development. I came to conclusion that I might as well
try each field on my own and incorporate my passions in each trial.

## Conclusion

In ICS 314, I hope to learn more software engineering practices and develop my profile. I want to practice and refine the skills that are needed to be a software engineer.
Looking through the ICS 314 topics, I am excited to try each module and hopefully one of the module will ignite a life-long journey.
